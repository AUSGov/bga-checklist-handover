Design Notes

Important functionality

  • The checklist needs to be a repeatable pattern. Content authors need to be able to create new checklists.
  • The user needs to be able leave and return to the checklist page and have their checkbox progress saved. Ideally the checkbox tick is saved in a cookie so user progress is still in place across sessions.
  • External links within the checklist items should open in the same tab.
  • The green tick for a step only displays when all the grey ticks in a step are done.
  • When a step is closed, any sub-steps underneath them are automatically closed too.
  • The "Must do" tags are required for all checkable items. The grey "Tip" items are not checkable and should not have this tag.
  • The blue "Email my link" button in the modal should use the design system pattern for a text box combined with primary button.
  • Text in the accordion items is styled as per the demo. The default styling for these boxes is the standard BGA body copy styles.
  • Links in accordion items should be purple once visited as per the standard BGA in-text link styles.

Desirable functionality

  • Ideally, when the user returns to the checklist page after clicking on a link the page retains where they were up to (e.g the accordion that was open is still open and the scrollpoint is the same).
